# IEEE SSCS Code-a-Chip Submission Checklist

Use this checklist before finalizing the NeuroSAR submission to the IEEE Solid-State Circuits Society (SSCS) Code-a-Chip Travel Grant program. Each item must be verified and checked before submission.

---

## 1. Code Completeness and Correctness

- [ ] **All notebooks execute without error from top to bottom** (restart kernel and run all cells)
  - [ ] `00_data_exploration.ipynb`
  - [ ] `01_train_pinn.ipynb`
  - [ ] `02_interactive_demo.ipynb`
  - [ ] `03_fom_exploration.ipynb`
  - [ ] `04_inverse_design.ipynb`
  - [ ] `05_spice_validation.ipynb`
- [ ] **No hardcoded absolute paths** in any notebook or script (use relative paths or `os.path.join`)
- [ ] **All imports are from packages listed in `requirements.txt`** — no hidden dependencies
- [ ] **Random seeds are fixed** in all notebooks and scripts that involve randomness
- [ ] **Data generation script runs end-to-end** with no errors: `python data/generate_data.py --n_samples 1000 --quick-test`
- [ ] **Training script completes at least one epoch** without error: `python training/train.py --epochs 1 --smoke-test`
- [ ] **SPICE integration module** loads without error even if ngspice is not installed (graceful import failure with informative message)

---

## 2. Dependencies and Environment

- [ ] **`requirements.txt` is complete and pinned** — all packages have version constraints
- [ ] **Environment installs cleanly** in a fresh virtual environment:
  ```bash
  python -m venv test_env && source test_env/bin/activate
  pip install -r requirements.txt
  jupyter lab --version  # confirm JupyterLab is available
  ```
- [ ] **Python version compatibility** is documented (Python 3.9+; tested on 3.9, 3.10, 3.11)
- [ ] **GPU is optional** — all code runs on CPU with acceptable performance
- [ ] **No proprietary PDK files** are required for the base demo (synthetic data path works without Sky130)
- [ ] **Optional dependencies** (ngspice, sky130) are documented in README and guarded with try/except in code

---

## 3. Documentation Quality

- [ ] **README.md is complete and renders correctly** on GitHub
  - [ ] All badge links are valid or clearly marked as placeholders
  - [ ] Quick start instructions are accurate and complete
  - [ ] Repository layout matches the actual directory structure
  - [ ] Citation block is filled in or marked as placeholder
- [ ] **`docs/overview.md` is present and complete**
- [ ] **`docs/physics_model.md` is present** with full derivations
- [ ] **`docs/training_plan.md` is present** with all hyperparameters
- [ ] **`docs/TECH_REP.md` is present** and reads as a coherent 2–3 page technical report
- [ ] **All notebooks have clear markdown cells** explaining each step — a reader unfamiliar with the codebase can follow along
- [ ] **All figures have axis labels, units, and titles**
- [ ] **All non-obvious code blocks have comments** explaining the physics or engineering rationale

## 4. Figures and Outputs

- [ ] **All figures are generated and saved** to `figures/` during notebook execution
  - [ ] `loss_convergence.png` — training/validation loss curves
  - [ ] `waveform_comparison.png` — PINN vs. ground truth side-by-side
  - [ ] `fom_heatmap.png` — Walden FoM across design space

## 5. Model Checkpoint

- [ ] **`models/checkpoints/best_model.pt` is present** OR a script to reproduce it from scratch in ≤ 30 minutes is documented
- [ ] **Checkpoint loads correctly** with the current model code:
  ```python
  model = NeuroSARPINN(config)
  ckpt = torch.load('models/checkpoints/best_model.pt', map_location='cpu')
  model.load_state_dict(ckpt['model_state_dict'])
  model.eval()
  # Forward pass sanity check
  out = model(sample_input)
  assert out.shape == (batch_size, 2)
  ```
- [ ] **Checkpoint metadata** (epoch, val loss, config) is inspectable and documented
- [ ] **Checkpoint size is reasonable** for submission (< 100 MB; the default ~270k-parameter model is ~3 MB)
- [ ] **If checkpoint is hosted externally** (e.g., Hugging Face Hub or Zenodo), the download script is tested and documented

## 6. Tests

- [ ] **All unit tests pass**: `pytest tests/ -v`
  - [ ] `tests/test_data_gen.py` — data shape, value ranges, physics sanity
  - [ ] `tests/test_losses.py` — loss values match hand-computed examples
  - [ ] `tests/test_model.py` — forward pass shapes, gradient flow
- [ ] **No test relies on trained model weights** (tests use random initialization)
- [ ] **Test suite completes in < 2 minutes** on CPU
